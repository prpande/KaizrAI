# Automation Conventions

- Automations orchestrate multiple prompts and tool calls
- Every automation run is tracked in `automation-runs.db`:
  - Start: `bash scripts/workspace-db.sh --action start-run --params '{"automation":"<name>"}'`
  - Complete: `bash scripts/workspace-db.sh --action complete-run --params '{"id":"<id>","steps_completed":"..."}'`
  - Fail: `bash scripts/workspace-db.sh --action fail-run --params '{"id":"<id>","error":"..."}'`
- Every step should have explicit fallback behavior on failure
- Automations should be idempotent where possible (running twice shouldn't duplicate data)
- Always start by recording the run, always end by completing/failing it
- Never fail silently — always tell the user what was skipped and why
