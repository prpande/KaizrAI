# Prompt Conventions

- Prompts are single-purpose, focused tasks
- Follow DRY: if multiple automations need the same logic, extract a prompt
- Reference schema docs in `schemas/` for database access patterns
- Always use the utility script for database operations, never raw SQL:
  `bash scripts/workspace-db.sh --action <action> --params '<json>'`
- Always read `.env` to resolve `WORKSPACE_DATA_DIR` before accessing data files
- Prompts should be testable in isolation — no dependency on prior conversation context
- Every prompt follows the standard format: Goal, Context, Instructions, Output, Changelog
